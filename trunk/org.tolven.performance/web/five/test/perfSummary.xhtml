[refresh].

Elapsed time is measured from the moment an HTTP request is received at the server until after the transaction and rendering have occurred for the page.

This display does not measure end-user response time, which is measured in the Tolven benchmark project. Many end-user interactions have no server involvement at all, such as moving back and forth between tabs and viewing a page that has already been fetched from the server. On the other hand, some complex pages require several server interactions. When this happens, the requests are handled in parallel and thus the elapsed time of all of the server requests could be much larger than what the user sees as response time. Therefore, rarely does server interactions equate to the number or duration of user interactions.
